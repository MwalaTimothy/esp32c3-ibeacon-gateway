───────────────────
 ✓ ESP32-C3 microcontroller (LOLIN C3 Mini recommended)
 ✓ USB-C cable (power + serial data)
 ✓ Computer running Windows 10+, macOS, or Linux
 ✓ WiFi network with internet access
 ✓ KDS server on your network (IP:port)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

QUICK SUMMARY
─────────────
Three ways to flash:
  1. Arduino IDE (beginner-friendly, graphical)
  2. PlatformIO (advanced, command-line)
  3. esptool.py (fastest for pre-built binaries)

We recommend Option 1 (Arduino IDE) for first-time users.


OPTION 1: Arduino IDE (Recommended)
────────────────────────────────────

STEP A: Install Arduino IDE
  • Go to: https://www.arduino.cc/en/software
  • Download installer for your OS
  • Install and launch Arduino IDE

STEP B: Add ESP32 Board Support
  • File menu → Preferences
  • Find: "Additional Board Manager URLs"
  • Paste URL: https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  • Click OK

  • Tools menu → Board Manager
  • Search: "esp32"
  • Click: "ESP32 by Espressif Systems"
  • Click "Install" (wait for download)
  • Close Board Manager

STEP C: Install NimBLE Library
  • Sketch menu → Include Library → Manage Libraries
  • Search: "NimBLE-Arduino"
  • Click: "NimBLE-Arduino by h2zero"
  • Select version 2.1.0 or newer
  • Click "Install"
  • Close Library Manager

STEP D: Prepare Your Code
  • Copy entire gateway_firmware_v2.1.ino source code
  • File → New sketch
  • Paste code into editor
  • File → Save As: "gateway_firmware_v2.1.ino"

STEP E: Configure Board Settings
  • Tools → Board → ESP32 Arduino → "LOLIN C3 Mini"
  • Tools → Port → Select your COM port (e.g., COM3, COM4)
  • Tools → Upload Speed → "460800"

STEP F: Flash Firmware
  • Sketch → Upload
  • Watch the bottom status bar for progress
  • Wait for: "Leaving... Hard resetting via RTS pin"
  • Success! ✅

STEP G: Verify Boot
  • Tools → Serial Monitor
  • Baud: 115200 (check dropdown bottom-right)
  • Press reset button on ESP32-C3
  • Should see boot messages:
      [INFO] Configuration window open for 180 seconds
      [INFO] Type 'HELP' for available commands


OPTION 2: PlatformIO (Advanced)
────────────────────────────────

  • Install PlatformIO extension in VS Code
  • Create new project for ESP32-C3
  • Copy source code into src/main.cpp
  • Click "Upload" button
  • Monitor serial output


OPTION 3: esptool.py (Pre-built Binary)
─────────────────────────────────────────

INSTALL esptool:
  • Open command prompt / terminal
  • Type: pip install esptool

FIND YOUR SERIAL PORT:
  Windows:
    • Device Manager → Ports (COM & LPT)
    • Look for "USB Serial" or "CH340"
    • Note the COM port (e.g., COM3)

  Mac:
    • Terminal: ls /dev/tty.*
    • Look for /dev/tty.usbserial

  Linux:
    • Terminal: ls /dev/ttyUSB*
    • Look for /dev/ttyUSB0

FLASH THE BINARY:
  Windows:
    esptool.py --chip esp32c3 --port COM3 --baud 460800 ^
      write_flash 0x0 gateway_firmware_v2.1.bin

  Mac/Linux:
    esptool.py --chip esp32c3 --port /dev/ttyUSB0 --baud 460800 \
      write_flash 0x0 gateway_firmware_v2.1.bin

EXPECTED OUTPUT:
  • esptool.py v4.X
  • Connecting...
  • Chip is ESP32-C3 (revision: 1)
  • Uploading ...
  • Verifying ...
  • Hard resetting via RTS pin
  • ✓ Done!

────────────────────────────────

  • Bottom-right: Check baud rate = 115200
  • Press reset button on ESP32-C3
  • You should see boot messages


PuTTY / Tera Term (Windows)
────────────────────────────
  • Download: https://www.putty.org/
  • Serial line: COM3 (your port)
  • Speed: 115200
  • Connect

Alternative Terminal (Windows)
  • Download: https://www.chiark.greenend.org.uk/~sgtatham/putty/
  • Or use: Windows Terminal (from Microsoft Store)


Terminal (Mac/Linux)
─────────────────────
  • screen /dev/ttyUSB0 115200
  • Press reset button on ESP32-C3
  • To exit: Ctrl+A, then Ctrl+D
  
  OR:
  • picocom /dev/ttyUSB0 -b 115200
  • Ctrl+A, Ctrl+X to exit

 SECTION 4: INITIAL CONFIGURATION (180-Second Window)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━��━━━━━━━━━━━━━━

⏰ CRITICAL: You have 180 seconds (3 minutes) to configure the gateway!
After 180 seconds, the configuration window closes automatically.

The LED on GPIO3 will blink slowly during this window.


STEP 1: View Current Configuration
──────────────────────────────────
Type in serial monitor:
  CONFIG

You will see:
  ========== GATEWAY CONFIGURATION ==========
  Gateway ID:        gateway-001
  Beacon UUID:       8a46676d-6348-4d50-a0e6-af5a508def40
  Target Major:      1
  KDS Server:        192.168.1.100:8080
  Scan Interval:     5000 ms
  Report Interval:   10000 ms
  WiFi SSID:         <not set>
  WiFi Password:     <not set>
  WiFi Status:       ✗ Disconnected
  Tracked Beacons:   0
  ===========================================


STEP 2: Configure WiFi
──────────────────────
FASTEST METHOD: One-liner setup + connect + save
Type:
  AT+WIFI=YourSSIDName,YourPassword

Example:
  AT+WIFI=Office,SecurePass123

Then wait for:
  [INFO] WiFi connected successfully
  [INFO] IP Address: 192.168.1.50
  [INFO] RSSI: -45 dBm

ALTERNATIVE METHOD: Three commands
Type:
  SET WIFI_SSID Office
  SET WIFI_PASS SecurePass123
  WIFI CONNECT

Wait for connection confirmation.


STEP 3: Verify WiFi Connection
───────────────────────────────
Type:
  WIFI STATUS

You should see:
  WiFi is connected
  SSID: Office
  IP: 192.168.1.50
  RSSI: -45 dBm


STEP 4: Configure KDS Server
─────────────────────────────
If your KDS server is NOT at 192.168.1.100:8080, update it:

Type:
  SET KDS_IP 192.168.1.100
  SET KDS_PORT 8080

(Replace with your actual KDS IP and port)


STEP 5: (Optional) Configure Gateway Identity
────────────────────────────────────────────────
If you want to change the gateway ID (default: "gateway-001"):

Type:
  SET GW_ID kitchen-1

(Use any descriptive name for your gateway)


STEP 6: Save Configuration
──────────────────────────
Type:
  SAVE

You will see:
  [INFO] Configuration saved to NVRAM

✓ All settings are now persistent (survive power cycles)


STEP 7: Wait for Automatic Exit
───────────────────────────────
After 180 seconds, you will see:
  [INFO] Configuration window closed
  [INFO] ✓ WiFi is connected - Beacon data will be sent to KDS
  [INFO] Gateway is now running. Listening for beacons...

✓ Gateway is now in operational mode!

STEP 1: Confirm Gateway is Listening
─────────────────────────────────────
In serial monitor, you should see:
  [INFO] Starting continuous BLE scan for beacons with Major: 1
  [INFO] ✓ WiFi is connected - Beacon data will be sent to KDS
  [INFO] Gateway is now running. Listening for beacons...


STEP 2: Trigger a Beacon Near the Gateway
──────────────────────────────────────────
If you have a companion BLE-iBeacon beacon device:
  • Power it on (or activate it)
  • Bring it within 3-10 meters of the gateway
  • Vibration should trigger the beacon (if SW-420 sensor)

The gateway should detect it within 1-5 seconds.


STEP 3: Monitor Serial Output
─────────────────────────────
You should see beacon detection:
  [2026-04-25 10:30:18] [BEACON] NEW - Major:1 Minor:5 RSSI:-62 dBm
  [2026-04-25 10:30:18] [INFO] Beacon tracking list:
    [1] Major:1 Minor:5 RSSI:-62 dBm

If you have a beacon app (nRF Connect):
  • Install on your phone
  • Search for iBeacon UUID: 8a46676d-6348-4d50-a0e6-af5a508def40
  • You should see beacon advertisements


STEP 4: List Tracked Beacons
────────────────────────────
Type in serial monitor:
  BEACONS

You will see:
  ========== TRACKED BEACONS ==========
  [1] Major:1 Minor:5 RSSI:-62 dBm | Seen:3 times | Uptime:5s
  ====================================

Each time the beacon is detected again, "Seen" count increases.


STEP 5: Verify HTTP POST to KDS Server
───────────────────────────────────────
Watch serial output for:
  [INFO] Sending beacon data to KDS server (192.168.1.100:8080)...
  [INFO] ✓ Beacon data sent to KDS successfully (HTTP 200)
  Payload: {"gateway_id":"kitchen-gate-1", ...}

This appears every 10 seconds (configurable).

If you see HTTP errors:
  [ERROR] HTTP POST failed: connection refused
  
Then KDS server is unreachable. Check:
  1. KDS server IP is correct: CONFIG
  2. KDS server is running
  3. Firewall allows port 8080 (or your port)


STEP 6: Check KDS Server Received Data
──────────────────────────────────────
On your KDS server, check logs for POST requests to /beacons endpoint.

You should see JSON like:
  {
    "gateway_id": "kitchen-gate-1",
    "major": 1,
    "minors_detected": [5],
    "rssi": [-62],
    "timestamp": "2026-04-25T10:30:00Z"
  }

✓ Integration complete!

PROBLEM: Serial monitor shows garbled text
───────────────────────────────────────────
CAUSE: Incorrect baud rate
SOLUTION:
  • Bottom-right of Arduino IDE: Set baud rate to 115200
  • Restart serial monitor


PROBLEM: Can't find COM port in Arduino IDE
────────────────────────────────────────────
CAUSE: USB driver missing or ESP32-C3 not detected
SOLUTION:
  1. Download CH340 driver:
     - Windows: https://www.wch.cn/downloads/CH341SER_EXE.html
     - Mac: https://www.wch.cn/downloads/CH341SER_MAC_ZIP.html
  2. Install driver
  3. Restart computer
  4. Restart Arduino IDE
  5. Tools → Port should now show your COM port


PROBLEM: Flashing fails with "Permission Denied"
─────────────────────────────────────────────────
CAUSE: Serial port locked or insufficient permissions
SOLUTION:
  Windows:
    • Restart Arduino IDE as Administrator
    • Try another USB port
    • Try another USB cable
  
  Mac/Linux:
    • sudo chmod 666 /dev/ttyUSB0
    • Or add user to dialout group


PROBLEM: WiFi won't connect
────────────────────────────
CAUSE: Incorrect SSID/password, or network incompatible
SOLUTION:
  1. Type: WIFI SCAN
  2. List your network in results?
     • YES: Check exact SSID spelling (case-sensitive)
     • NO: Router might be 5 GHz only (ESP32-C3 needs 2.4 GHz)
  3. Type: WIFI DISCONNECT
  4. Wait 2 seconds
  5. Type: AT+WIFI=YourSSID,YourPassword
  6. Wait for connection message


PROBLEM: WiFi says connected but no beacons reported to KDS
──────────────────────────────────────────────────────────
CAUSE: KDS server unreachable
SOLUTION:
  1. From computer: ping 192.168.1.100 (your KDS IP)
     • Pings fail? Gateway and KDS on different networks
     • Try setting correct IP: SET KDS_IP X.X.X.X
  2. Check KDS firewall allows port 8080
  3. Verify /beacons endpoint exists on KDS
  4. Check gateway logs: Type CONFIG and verify KDS settings


PROBLEM: Beacons not detected (BEACONS list empty)
─────────────────────────────────────────────────
CAUSE: UUID or Major mismatch, or no beacon in range
SOLUTION:
  1. Enable debug: BLE DEBUG ON
  2. Bring beacon very close (< 1 meter)
  3. Check serial output for per-packet info
  4. If no packets: Beacon not advertising
  5. If packets visible:
     - Verify UUID matches: SET UUID <correct-uuid>
     - Verify Major matches: SET MAJOR <beacon-major>
     - Or use wildcard: SET MAJOR 65535
  6. Type: SAVE and reset


PROBLEM: LED not blinking during config window
───────────────────────────────────────────────
CAUSE: LED not connected, wrong polarity, or damaged
SOLUTION:
  • If using external LED: Check polarity (long leg = +)
  • Try replacing LED
  • LED not critical for operation

 SECTION 7: ADVANCED CONFIGURATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

After basic setup, you can fine-tune the gateway:


TRACK ALL BEACONS (Wildcard Major)
──────────────────────────────────
To track every beacon regardless of Major:
  SET MAJOR 65535
  SAVE

Now reports include all detected beacon majors.


INCREASE SCAN DEPTH (Noisy Environment)
────────────────────────────────────────
For deeper BLE discovery:
  SET SCAN_MS 10000      # Scan 10 seconds per cycle
  SAVE


REDUCE REPORT FREQUENCY (Lower KDS Load)
─────────────────────────────────────────
To report less frequently:
  SET REPORT_MS 30000    # Report every 30 seconds
  SAVE

Tradeoff: Slower updates to KDS, but lower network traffic.


ENABLE PER-PACKET DEBUG (Inspect Beacons)
──────────────────────────────────────────
To see every iBeacon received:
  BLE DEBUG ON

Output:
  [BLE-DBG] iBeacon mfg:004C major:1 minor:5 rssi:-62

Then:
  BLE DEBUG OFF
